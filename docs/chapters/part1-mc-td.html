5  Monte Carlo Methods and Temporal-Difference Learning

Dynamic programming assumes that the environment model is known. In many realistic problems, that assumption is too strong. The agent does not know the transition probabilities in advance, and it may not know the reward structure either. What it does have is experience.

Monte Carlo methods and temporal-difference learning answer the same broad question:

How can an agent learn value estimates directly from sampled interaction?

These two families of methods are fundamental. They show how reinforcement learning can move from planning with a known model to learning from data.

5.1 Monte Carlo Methods

Monte Carlo methods learn from complete episodes. The core idea is straightforward:

  1. Run an episode.
  2. Observe the return that actually occurred.
  3. Use that return as a target for value estimation.

If a state \(s\) appears in an episode, we can estimate its value by averaging the returns observed after visiting that state. Over many episodes, these averages can converge to the true value under the policy being followed.

This approach has an appealing simplicity. It does not require a model, and it does not bootstrap from current estimates. It waits until the return is fully observed.

For example, if the return following a visit to state \(s\) is \(G_t\), then a Monte Carlo estimate of the state value may be updated toward that sampled return:

\[ V(s) \leftarrow V(s) + \alpha \left(G_t - V(s)\right) \]

This looks very similar to later update rules. The key difference lies in the target. In Monte Carlo learning, the target is a full sampled return.

In the maintained code for this book, the cleanest tabular Monte Carlo reference is:

  1. examples/tabular/monte_carlo_prediction.py

The important thing to notice in that file is not just that it samples episodes. It also stores complete trajectories and then walks backward through them to compute returns. That backward pass makes the full-return target explicit in code.

5.2 First-Visit and Every-Visit Monte Carlo

There are different ways to count returns for a state within an episode.

In first-visit Monte Carlo, only the first occurrence of a state in an episode is used for updating that state’s estimate. In every-visit Monte Carlo, every occurrence is used.

Both approaches can work. The difference is mostly about how samples are collected and correlated. In practice, the main idea remains the same: use complete returns from actual trajectories.

The maintained example currently uses a first-visit style update. In run_first_visit_mc_prediction(...), the code checks whether the current state has already appeared earlier in the same episode before committing the update. That one implementation detail is the practical difference between first-visit and every-visit Monte Carlo.

5.3 Strengths and Weaknesses of Monte Carlo Learning

Monte Carlo methods have several strengths:

  1. They are conceptually simple.
  2. They do not require a model.
  3. They use unbiased sampled returns as targets.

But they also have limitations:

  1. They require episodes to terminate, or at least to be truncated in a meaningful way.
  2. They may learn slowly when episodes are long.
  3. Their return estimates can have high variance.

That high variance is important. If rewards are delayed or noisy, the complete return can fluctuate greatly from one episode to the next. This makes value estimation unstable or slow.

5.4 Temporal-Difference Learning

Temporal-difference learning, usually abbreviated TD learning, takes a different approach. Instead of waiting for the full return, it updates estimates using partially observed experience plus current value estimates.

The simplest case is TD(0):

\[ V(S_t) \leftarrow V(S_t) + \alpha \left(R_{t+1} + \gamma V(S_{t+1}) - V(S_t)\right) \]

This update uses a bootstrapped target:

\[ R_{t+1} + \gamma V(S_{t+1}) \]

Rather than waiting until the end of the episode, TD learning says:

Use the immediate reward plus the current estimate of the next state as a target right now.

This idea is one of the deepest in reinforcement learning. The target is not a full observed outcome. It is part observation, part estimate.

In the maintained code, the cleanest TD(0) example is:

  1. examples/tabular/td0_prediction.py

The key function is run_td0_prediction(...). If you compare it with the Monte Carlo file, the contrast is instructive. The Monte Carlo script stores a whole episode and updates later. The TD script updates immediately after each transition. That single structural difference captures a large part of the conceptual gap between Monte Carlo and TD learning.

5.5 The TD Error

The quantity

\[ \delta_t = R_{t+1} + \gamma V(S_{t+1}) - V(S_t) \]

is called the temporal-difference error.

It measures how surprising the transition was relative to the current value estimate. If the outcome was better than expected, the TD error is positive and the estimate should increase. If the outcome was worse than expected, the TD error is negative and the estimate should decrease.

This error signal later becomes important far beyond basic prediction. Actor-critic methods use related ideas. Advantage estimation is closely connected to it. Many practical deep reinforcement learning algorithms still depend on TD-style targets.

5.6 Monte Carlo vs. Temporal-Difference Learning

Monte Carlo and TD methods both learn from experience, but they make different trade-offs.

Monte Carlo methods:

  1. Wait for full returns.
  2. Do not bootstrap.
  3. Can have high variance.
  4. Are naturally episodic.

Temporal-difference methods:

  1. Update after each step.
  2. Bootstrap from current estimates.
  3. Usually have lower variance.
  4. Can work naturally in continuing tasks.

A common summary is:

Monte Carlo methods wait longer but use more complete targets. TD methods learn earlier but rely on estimates that may themselves be wrong.

This is a bias-variance trade-off. Monte Carlo targets are often less biased but more variable. TD targets are often more biased but less variable and more data-efficient.

That trade-off is not merely theoretical. You can see it in the code organization of the maintained examples:

  1. examples/tabular/monte_carlo_prediction.py
  2. examples/tabular/td0_prediction.py

The Monte Carlo script has to wait for trajectory completion and then reconstruct returns. The TD script can learn online, one transition at a time. The code itself shows why TD methods tend to feel more incremental and more data-efficient in practice.

5.7 Bootstrapping as a Core Idea

Bootstrapping means updating an estimate using another estimate. This may seem risky at first, because errors can propagate. But it is also what makes learning efficient. Instead of waiting for full trajectories, the agent can improve estimates online, one transition at a time.

Bootstrapping is also one of the reasons reinforcement learning can become unstable when combined with function approximation. Later in the book, when we study DQN and actor-critic methods, we will see that bootstrapping remains powerful but must be handled carefully.

5.8 Eligibility Traces and the Space Between MC and TD

Monte Carlo and TD(0) are often presented as separate families, but they are really endpoints of a spectrum. Methods based on eligibility traces, such as TD(\(\lambda\)), interpolate between them.

When \(\lambda\) is closer to 1, the method behaves more like Monte Carlo, relying more on long returns. When \(\lambda\) is closer to 0, it behaves more like TD(0), relying more on one-step bootstrapping.

This spectrum is important because it shows that reinforcement learning methods are often connected by design trade-offs rather than by sharp boundaries.

This book does not yet maintain a full TD(\(\lambda\)) code path in examples/, so the point here should be understood conceptually. It is included because it explains why Monte Carlo and TD(0) should not be thought of as isolated islands. They are endpoints of a family of return-construction choices.

5.9 From Prediction to Control

So far, we have mostly discussed value estimation for a fixed policy. But reinforcement learning is usually about control, not just prediction. We want the agent to improve its behavior while learning.

That requires one more step:

The agent must use value estimates to choose better actions.

The next chapter studies this transition. We will move from value prediction to classical control methods, including exploration strategies, SARSA, and Q-learning.
