6  Classical Control Methods

Prediction methods estimate how good a policy is. Control methods go one step further: they try to find a good policy.

In reinforcement learning, this usually means that the agent must both learn and act at the same time. It cannot wait until all uncertainty disappears. It must make decisions while its estimates are still imperfect. That makes control fundamentally harder than pure policy evaluation.

This chapter introduces the basic logic of classical model-free control. The key themes are exploration, action-value estimation, and iterative policy improvement through experience.

6.1 Exploration and Exploitation

One of the first problems in control is the exploration-exploitation trade-off.

If the agent always chooses the action that currently looks best, it may miss better actions that have not been tried enough. If it explores too much, it may waste reward on poor choices and never settle into strong behavior.

This trade-off is not a minor technical detail. It is one of the central design problems of reinforcement learning.

Many classical methods begin with a simple exploration rule such as (\(\epsilon\))-greedy action selection:

  1. With probability (1 - ), choose the action with the highest estimated value.
  2. With probability (), choose a random action.

This rule is crude, but it is easy to implement and often works well enough in small problems. It also appears in many toy implementations, including classic Q-learning examples.

In the maintained codebase, the simplest places to see this idea are:

  1. examples/tabular/sarsa.py
  2. examples/tabular/q_learning.py

Both rely on an \(\epsilon\)-greedy action selector in examples/tabular/control_utils.py. This is a good example of a principle worth carrying from theory into code: exploration is not an optional outer wrapper around learning. It is part of what determines the data distribution, and therefore part of what the agent learns.

6.2 Action-Value Control

For control, action values are often more useful than state values. If we estimate (Q(s, a)), then we can select actions directly by comparing action values within the current state.

This is the basic value-based control idea:

  1. Estimate action values.
  2. Choose actions using those estimates.
  3. Update the estimates from experience.
  4. Repeat.

Once action values become accurate enough, greedy action selection can define a strong policy.

6.3 On-Policy and Off-Policy Learning

Before introducing SARSA and Q-learning, we need one important distinction.

An on-policy method learns about the policy it is currently using to act. An off-policy method learns about one policy while following another.

This distinction matters because the behavior policy may include exploration, while the target policy may be greedy or nearly greedy. Off-policy methods are often more flexible, but they can also become less stable, especially once function approximation is introduced.

SARSA is the classical on-policy example. Q-learning is the classical off-policy example.

6.4 SARSA

SARSA is named after the sequence of variables used in its update:

\[ (S_t, A_t, R\_{t+1}, S\_{t+1}, A\_{t+1}) \]

The update rule is:

\[ Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha \left[R_{t+1} + \gamma Q(S_{t+1}, A_{t+1}) - Q(S_t, A_t)\right]\]

This is an on-policy update because the next action (A_{t+1}) is the action actually chosen by the current behavior policy, which may include exploration.

That detail has an important consequence. SARSA learns the value of behaving according to the exploration-aware policy currently in use. If exploratory actions can be dangerous, SARSA may learn more conservative behavior than a purely greedy method.

This makes SARSA especially useful for understanding how learning depends not only on the update rule, but also on the behavior policy generating the data.

The maintained implementation is:

  1. examples/tabular/sarsa.py

If you read run_sarsa(...), the most important line conceptually is the one that selects next_action from the current exploratory policy before building the TD target. That is the code-level meaning of “on-policy” in this chapter. The target is tied to what the agent actually does next, not to a hypothetical greedy action it did not take.

6.5 Q-Learning

Q-learning uses a different target:

\[ Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha \left[R_{t+1} + \gamma \max_a Q(S_{t+1}, a) - Q(S_t, A_t)\right]\]

The max over next actions means the update targets the value of the greedy action, regardless of which action is actually taken next by the behavior policy. That is why Q-learning is off-policy.

Conceptually, Q-learning says:

Act with exploration if needed, but learn as if you were going to behave greedily from the next state onward.

This is a powerful idea. It allows exploratory behavior to collect data while still pushing estimates toward the greedy optimal action-value function (Q^*).

Q-learning is one of the most influential algorithms in reinforcement learning. Its basic structure later reappears in DQN, where the tabular action-value table is replaced by a neural network.

The maintained implementation is:

  1. examples/tabular/q_learning.py

Here the code-level contrast with SARSA is sharp. In run_q_learning(...), the target uses max(q_table[next_state].values()) rather than the value of the next exploratory action actually chosen. That single difference is the operational meaning of “off-policy” in this setting.

6.6 SARSA vs. Q-Learning

SARSA and Q-learning often look very similar in code. Both update action-value estimates from one-step transitions. Both are easy to implement in tabular environments. Both can use ()-greedy exploration.

But their learning targets differ:

  1. SARSA uses the next action actually chosen by the current policy.
  2. Q-learning uses the best estimated next action, whether or not it is chosen.

This difference can lead to meaningfully different behavior. In risky environments, SARSA may learn safer policies because it accounts for the exploratory actions that the agent really takes. Q-learning may learn more aggressive policies because it evaluates the greedy target rather than the exploratory behavior policy.

In this project, both maintained scripts use a negative-grid variant with step cost. That is a sensible foundation example because it makes the behavioral difference easier to see. The point is not only whether the final policy reaches the goal. The point is how the learning target changes the kind of path the agent prefers while exploration is still part of the process.

6.7 Convergence and Practical Assumptions

In finite tabular settings, classical convergence results are available under assumptions such as:

  1. All relevant state-action pairs are visited sufficiently often.
  2. Learning rates are chosen appropriately.
  3. Exploration does not vanish too early.

These assumptions matter because reinforcement learning is only as good as the data it sees. If the agent never explores a useful action, it cannot learn its value.

At the same time, convergence guarantees in the tabular case should not create false confidence about larger problems. Once we move to function approximation and neural networks, many clean guarantees become weaker or disappear entirely.

6.8 A Note on Historical Implementations

Many older reinforcement learning tutorials implement Q-learning with small lookup tables, simple ()-greedy exploration, and lightweight loops over episodes. Those examples are still excellent for understanding the algorithmic core.

For example, a compact tabular Q-learning implementation usually contains the following ingredients:

  1. A table storing (Q(s, a)).
  2. An ()-greedy action selector.
  3. A one-step TD update toward a bootstrap target.
  4. Repetition over many episodes.

That structure is still worth studying. The main limitation is not the logic of the algorithm, but the scale of the representation. Once state spaces become large or continuous, lookup tables stop being practical.

In the maintained code for this book, that compact structure is still present, just cleaned up and reorganized:

  1. examples/tabular/control_utils.py
  2. examples/tabular/sarsa.py
  3. examples/tabular/q_learning.py

These files are deliberately small enough that you can trace the entire control loop from exploration to update target to policy extraction without jumping across a large framework.

6.9 Why Classical Control Still Matters

Classical control methods are not just historical stepping stones. They teach several durable lessons:

  1. Value estimation and action selection are tightly linked.
  2. Exploration strategy changes what the agent learns.
  3. On-policy and off-policy learning are meaningfully different.
  4. Bootstrapping can support efficient control.

These lessons carry directly into deep reinforcement learning. DQN inherits the off-policy value-learning structure of Q-learning. Policy-gradient and actor-critic methods respond, in part, to the limitations of value-based control in large or continuous spaces.
