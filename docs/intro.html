1  Introduction

Reinforcement learning studies how an agent can learn to make decisions through interaction with an environment. Unlike supervised learning, where a model is trained from labeled input-output pairs, reinforcement learning is about sequential action. The learner is not told the correct action at every step. Instead, it acts, observes the consequences, and receives rewards that may be delayed, noisy, or sparse.

This delayed-feedback setting is what makes reinforcement learning both powerful and difficult. A decision that looks bad in the short term may create a better future state. A decision that gives immediate reward may block better long-term outcomes. Because of this, reinforcement learning is not only about prediction. It is about strategy.

At a high level, the field asks a simple question:

How should an agent act so that the total long-term reward is as large as possible?

That question appears in many forms. A game-playing agent chooses moves. A robot chooses control signals. A recommendation system chooses which item to show. A trading system chooses whether to buy, sell, or wait. In each case, the system must act under uncertainty, and each action can influence what happens next.

1.1 Why Start with Classical Reinforcement Learning?

Deep reinforcement learning receives most of the attention today because it combines neural networks with reinforcement learning objectives. That combination made it possible to learn from high-dimensional observations such as images, and it led to famous results in Atari games, Go, robotics, and large-scale decision systems.

However, deep reinforcement learning is much easier to misunderstand if we skip the classical foundations. DQN is still built on Q-learning. Actor-critic methods still depend on value estimation. Advantage functions still come from the same logic as return estimation and baseline correction. If the foundations are weak, deep methods can look like a collection of engineering tricks rather than a coherent family of ideas.

For that reason, this book begins with:

  1. The agent-environment framework.
  2. Markov decision processes.
  3. Value functions and Bellman equations.
  4. Dynamic programming.
  5. Monte Carlo methods.
  6. Temporal-difference learning.
  7. Classical control methods such as SARSA and Q-learning.

Only after that do we move into function approximation and deep reinforcement learning.

1.2 Why Deep Reinforcement Learning Is Hard

The move from classical reinforcement learning to deep reinforcement learning is not just a change of model size. It changes the training dynamics in important ways.

In tabular reinforcement learning, each state or state-action pair can be stored explicitly. Updates are local and often interpretable. In deep reinforcement learning, a neural network shares parameters across many states. This gives us generalization, but it also introduces instability. A single gradient update can change predictions for many states at once. Bootstrapping, off-policy learning, and function approximation can interact in unstable ways. This is one reason techniques such as replay buffers, target networks, normalization, clipping, trust regions, and entropy regularization became so important in practice.

Another difficulty is evaluation. In supervised learning, a validation set often provides a relatively direct signal of progress. In reinforcement learning, performance can vary greatly across random seeds, reward scales, exploration strategies, environment wrappers, and implementation details. Two systems that look similar on paper may behave very differently during training.

1.3 How to Read This Book

This book is organized as a gradual progression.

The first part builds the conceptual and mathematical foundations of reinforcement learning. The second part bridges classical methods and deep methods. The third part focuses on practical deep reinforcement learning, including stability, debugging, and implementation judgment.

When reading, it helps to keep three questions in mind:

  1. What is being estimated?
  2. What is being improved?
  3. What assumptions make the method work?

Those three questions are useful almost everywhere in reinforcement learning. A value-based method estimates values and improves a policy indirectly. A policy-based method improves the policy directly but may still estimate returns or advantages. An actor-critic method does both. If you keep track of those roles, the algorithm landscape becomes much easier to organize.

1.4 A Note on the Example Code

The main code path for this book is the maintained examples/ directory. That is the codebase the chapters are primarily written against. It contains the tabular examples, the function-approximation bridge examples, the maintained PyTorch DQN path, and the current policy-gradient baseline used by the project.

Older reinforcement learning examples can still have value, especially when a compact implementation exposes an algorithmic skeleton clearly. But the software stack has changed, and many older tutorials assume earlier Gym APIs, TensorFlow 1.x, or older training conventions. In this book, that older style of material is used only as conceptual background:

  1. To illustrate the shape of an algorithm.
  2. To compare tabular and neural approaches.
  3. To discuss how modern implementations differ.

Whenever an implementation detail is outdated, the text will say so clearly rather than presenting old patterns as current best practice. The general rule is simple:

  1. examples/ is the maintained code path to read with the book.
  2. Older tutorial-style implementations are background material, not the primary source of truth for the text.

By the end of the book, the reader should not only know what major reinforcement learning algorithms do, but also understand why they were developed, when they fail, and how they connect to the broader logic of sequential decision-making.
